# Conduit â€” Mac app (React Native macOS)

The Mac side of Conduit. It discovers the phone over Bonjour, connects as a WebSocket client,
pairs, and drives the feature UI. All app logic lives in `src/` and `App.tsx`; the native macOS
Xcode project is generated by tooling (see below) and is intentionally not committed.

## What's here

```
App.tsx                     sidebar shell + screen routing
index.js                    react-native-macos entry point
src/protocol/types.ts       copy of the shared wire types
src/util/                   encoding (base64/utf8/uuid), sha256/hmac, persistent store
src/net/                    Discovery (Bonjour), ConnectionManager, binary codec, pairing proof
src/features/fileTransfer   chunked send/receive over the connection
src/hooks.ts                React bindings over the singletons
src/ui/                     Devices, Files, Messages, Notifications, Screen screens
```

The crypto, base64, UUID, binary-frame, and pairing-proof code in `src/util` + `src/net` is
verified byte-compatible with the Android side and type-checks under `strict`.

## Prerequisites

- macOS with **Xcode 16+** and **CocoaPods** (`sudo gem install cocoapods` or `brew install cocoapods`).
- Node 18+ and a working React Native macOS environment.

## Generate the native macOS project, then run

This repo ships the JS/TS app but not the generated `macos/` Xcode workspace. Create it once:

```bash
cd mac-app
npm install

# Scaffold the native macOS host into ./macos for this existing project:
npx react-native-macos-init

# Install native pods (zeroconf, fs, document-picker, clipboard autolink here):
npm run pods            # = cd macos && pod install

# Run it:
npm run macos
```

Type-check anytime with `npm run tsc`.

> **Xcode 16+ / clang note:** RN 0.76 pins `fmt` 11.0.2, which newer clang rejects with
> `call to consteval function â€¦ is not a constant expression`. fmt's `base.h` *unconditionally*
> `#define`s `FMT_USE_CONSTEVAL` (no `#ifndef` guard), so a `-D` flag can't override it. The
> generated `macos/Podfile` therefore has a `post_install` hook that rewrites that header to force
> the non-consteval path. If you regenerate the Podfile, re-add that hook.

## Native-module caveats (read before filing build issues)

macOS support in community modules varies. If a module fails to build on macOS:

- **react-native-zeroconf** â€” used for discovery. If it doesn't resolve on your RN-macOS version,
  swap in a thin native `NSNetServiceBrowser` module, or temporarily hard-code the phone's
  `host:port` in `DevicesScreen` to test the rest of the stack.
- **react-native-document-picker** â€” used only to choose a file to send. If unavailable on macOS,
  replace `pickAndSend` with an `NSOpenPanel`-backed native module; the transfer engine itself
  doesn't depend on it.
- **react-native-fs** â€” used for chunked disk read/write. Required for file transfer.

The connection core (discovery aside) uses only RN's built-in `WebSocket`, so pairing, heartbeat,
status, and the protocol work without extra native code.

## Known simplifications (foundation, not production)

- `ws://` + token auth today; `wss://` + cert pinning is the documented hardening step.
- Outgoing transfers read the whole file in 64 KB chunks but don't apply socket backpressure.
- Messages / Notifications screens are scaffolds tied to the Android scaffold handlers.
- Screen mirroring is a placeholder (see the Screen tab and `protocol/README.md`).
